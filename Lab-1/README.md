## 1. Purpose of Lab 1

Lab 1 has three connected goals.

First, every student must get access to the course GitHub repository and complete the required tool setup on their own machine.

Second, every student must understand the individual Python homework and how to complete and submit it properly.

Third, students begin the first stage of product thinking by brainstorming software ideas, first alone, then with teammates.

This lab mixes **individual work** and **team work**. The README must be read in order.

---

## 2. What Happens in Lab 1

Lab 1 is organized into three parts.

### Part 1, GitHub and tool setup
Students set up access to the course repository, prepare their machines, install the required tools, and confirm that they can run Python code locally.

### Part 2, individual Python homework briefing
Students review the individual homework, understand the requirements, inspect the starter code, and learn what they must submit on their own.

### Part 3, idea brainstorming
Students begin the ideation stage for the course project work. This starts as individual thinking and then moves into team discussion and synthesis.

---

## 3. Individual Work vs Team Work

This lab contains both.

### Individual work
Each student must:

- access the course GitHub repository
- complete local setup on their own machine
- understand the Python environment and starter files
- complete the individual Python homework
- brainstorm initial software ideas on their own before team discussion

### Team work
Students will also:

- discuss potential problem spaces
- compare individual ideas
- identify promising directions
- begin shaping shared team thinking for later project work

### Rule for Lab 1
Do not confuse the individual Python homework with the team brainstorming activity.

The Python homework is individual.

The brainstorming section includes both individual and team work.

---

## 4. Main Learning Outcomes

By the end of Lab 1, students should be able to:

1. access and navigate the course GitHub repository
2. set up the required tools for the course
3. create and activate a Python virtual environment
4. understand the structure of the starter code and support files
5. explain the individual Python homework requirements
6. generate initial software or product ideas individually
7. discuss and refine ideas in a team setting

---

## 5. Lab 1 Flow

The class runs in this order.

### Step 1, GitHub setup
Students join or access the course GitHub organization and repository. They confirm that they can open the correct lab folder and see the required files.

### Step 2, tool setup
Students prepare the required local environment. This includes Python, virtual environment setup, package installation, and other required software for the lab.

### Step 3, individual homework explanation
The instructor walks through the individual Python homework, including required files, starter code, expectations, and submission rules.

### Step 4, individual brainstorming
Each student thinks independently about software ideas, user problems, or opportunity spaces before speaking as a team.

### Step 5, team brainstorming
Students discuss their ideas in groups, compare directions, and begin identifying stronger concepts for later work.
